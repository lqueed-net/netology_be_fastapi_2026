Мы разработаем демонстрационное FastAPI приложение, которое реализует CRUD для сущности "Item" (товар) с использованием асинхронной SQLAlchemy и PostgreSQL. Приложение будет построено с учётом принципов SOLID, что обеспечит модульность, тестируемость и лёгкость расширения.


1. Настройка базы данных (database.py)
Создаём движок, фабрику асинхронных сессий и базовый класс для моделей.

2. Модель данных (models.py)
Определяем SQLAlchemy модель Item.

3. Схемы Pydantic (schemas.py)
Создаём схемы для запросов (создание, обновление) и ответов.

4. Репозиторий (repositories.py)
Абстрактный класс и конкретная реализация для работы с Item. Репозиторий инкапсулирует все запросы к БД, что соответствует принципу единственной ответственности.

5. Сервисный слой (services.py)
Сервис содержит бизнес-логику и зависит от абстрактного репозитория (Dependency Inversion). Здесь можно добавить валидацию, расчёты, вызов внешних API и т.д.

6. Эндпоинты FastAPI (routers/items.py)
Роутер получает сессию из зависимости, создаёт репозиторий и сервис, затем вызывает методы сервиса.

7. Главный файл приложения (main.py)
Собираем всё вместе, подключаем роутер и создаём таблицы при запуске (для демонстрации).

7. Кеш (cache.py)




    Single Responsibility:
models.Item – только определение модели.
schemas.* – только валидация ввода/вывода.
repositories.ItemRepository – только работа с БД.
services.ItemService – только бизнес-логика.
routers.items – только маршрутизация HTTP.

    Open/Closed:
Классы открыты для расширения, но закрыты для модификации. Например, можно создать новый репозиторий для другого типа хранилища, не меняя сервис.

    Liskov Substitution:
Любая реализация AbstractItemRepository может быть подставлена в ItemService без нарушения работы сервиса.


    Interface Segregation:
Интерфейс AbstractItemRepository содержит только необходимые методы для работы с Item.

    Dependency Inversion:
Сервис зависит от абстракции AbstractItemRepository, а не от конкретного класса. Репозиторий передаётся через конструктор (DI).
